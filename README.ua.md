## Що таке Forgeline?
**Forgeline** — це плагін для **Claude Code**. Уявіть, що це ваш особистий "менеджер з найму" ШІ-розробників.
Замість того, щоб годинами писати налаштування вручну, ви запускаєте одну команду, відповідаєте на прості запитання, і Forgeline автоматично створює для вас цілу команду ШІ-агентів. Вони розуміють ваш код, знають правила проєкту та готові до роботи. Це налаштовує систему за 5 хвилин замість 4 годин.

## Що потрібно для старту?
1. **Встановлений Claude Code.** (Ви можете встановити його через звичайний термінал командою `npm install -g @anthropic-ai/claude-code`).
2. **Платна підписка Claude.** (Claude Code працює лише за наявності активного акаунту Anthropic і доступу).
3. **Папка з проєктом.** Запускайте Claude Code саме в тій папці, де лежить (або буде лежати) ваш код.

## Крок 1: Встановлення
Відкрийте Claude Code (введіть команду `claude` у вашому терміналі) та по черзі запустіть ці дві команди, щоб завантажити та активувати плагін:
```bash
/plugin marketplace add nikita-voloshyn/forgeline
/plugin install nikita-voloshyn/forgeline
```

## Крок 2: Базове налаштування
Введіть у Claude Code команду:
```bash
/setup-agents
```
Forgeline проведе вас через 8 простих інтерактивних кроків. Це безпечно – жоден файл не буде створено до останнього кроку.
1. **Аналіз проєкту:** ШІ просканує код і спитає, чи правильно він визначив мову програмування та фреймворки.
2. **Підхід до розробки:** Виберіть стиль роботи (наприклад, Ітеративний або з фокусом на Тести).
3. **Агенти:** Він запропонує команду ШІ (фронтенд-агент, бекенд-агент тощо).
4-7. **Навички, плагіни, хуки, дозволи:** ШІ запропонує найкращі інструменти доступу для вашого проєкту. Якщо сумніваєтесь — можете просто погоджуватися із запропонованим.
8. **Підтвердження:** Ви побачите фінальний план. Якщо ви згодні, він згенерує понад 50 файлів для створення вашої нової ШІ-команди.

## Крок 3: Щоденна розробка
Коли команда налаштована, створення кожної нової фічі займає 3 прості кроки:

1. **Планування (Plan):**
```bash
/plan "Зроби сторінку авторизації"
```
*Що це робить:* ШІ глибоко аналізує завдання та розбиває його на дрібні чіткі кроки.

2. **Призначення (Assign):**
```bash
/assign
```
*Що це робить:* ШІ розподіляє ці кроки між правильними агентами (наприклад, бекенд-агенту дасть налаштувати базу даних, а фронтенд-агенту — зробити дизайн).

3. **Виконання (Execute):**
```bash
/execute
```
*Що це робить:* Команда спеціалізованих ШІ-агентів починає самостійно та по черзі писати код, перевіряти помилки та тести. Залишається просто дочекатися завершення!
